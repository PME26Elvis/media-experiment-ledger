## Reading the cards

Each row represents one controlled cohort. Images and videos are always separate media types, even when they share a prompt ID. The global index includes cohorts from every published `media-exp-*` Release, even when the latest upload did not contain that prompt.

Use the **Media** selector to show images, videos, or both.

### Image cards

Image cohorts are keyed by identical `prompt_id`, model, and appearance-relevant settings. A primary JPEG summarizes temporal anchors, an extended sheet selects up to 16 temporal quantiles, and full contact sheets contain every verified byte-unique image at 16 samples per page.

### Video cards

Video cohorts are keyed by identical `prompt_id`, model, and non-random generation settings. Random seed remains visible in sidecars but is excluded from the cohort key so repeated free-API runs can be compared.

The primary GIF synchronizes all tiles from `t=0`, freezes short videos on their last frame, and uses contain/letterbox without cropping. Complete keyframe pages contain 10%, 50%, and 90% frames for every verified byte-unique video.

Only the small Release-note highlight set is mirrored as inline web previews. Every cohort links to the deterministic Atlas bundle that contains its prompt. Image and video bundles are separate and each contains up to 15 prompt IDs.

## Validation

Images must match a ZIP `images` member and pass Pillow decoding. Videos must match a ZIP `videos` member, pass `ffprobe`, expose valid duration and dimensions, and decode successfully near the start, midpoint, and end through FFmpeg. Both media types use exact SHA-256 deduplication.

## Publication contract

Raw experiment Releases remain immutable. The common multi-date publisher dispatches one global rebuild after the whole batch succeeds. Derived tags use `media-analysis-all-<dataset-fingerprint>-vN`.

The companion Release asset list is ZIP-only:

- grouped image bundles, up to 15 prompt IDs each;
- grouped video bundles, up to 15 prompt IDs each;
- metadata and offline-gallery packages;
- multipart complete packages below the configured asset boundary.

Release Notes may display versioned JPEG and GIF previews, but those previews are served from repository paths rather than uploaded as naked Release assets.

See the [shared/image Atlas specification](https://github.com/PME26Elvis/media-experiment-ledger/blob/main/docs/PROMPT_REPEATABILITY_ATLAS.md) and [video Atlas production contract](https://github.com/PME26Elvis/media-experiment-ledger/blob/main/docs/VIDEO_REPEATABILITY_ATLAS.md).
