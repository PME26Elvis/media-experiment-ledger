## Tool inventory

<div class="tool-card-grid">
  <article class="tool-card"><code>agnes_media_harvester.py</code><p>Calls image/video APIs at conservative intervals and writes structured run metadata.</p><ul><li>Loads YAML and prompt banks</li><li>Resumes from state</li><li>Writes outputs/errors JSONL</li><li>Downloads completed media</li></ul></article>
  <article class="tool-card"><code>tools/publish_from_archive.py</code><p>ZIP-first ingress wrapper for Codespaces.</p><ul><li>Validates ZIP64 structure</li><li>Rejects traversal and unsafe links</li><li>Checks disk capacity</li><li>Finds the results root</li></ul></article>
  <article class="tool-card"><code>tools/input_snapshot.py</code><p>Stores a large input archive immediately and promotes it later.</p><ul><li>Byte-exact 1.8 GiB splitting</li><li>Part/source SHA-256</li><li>Restore and promote modes</li><li>Latest-tag resolution</li></ul></article>
  <article class="tool-card"><code>tools/publish_results.py</code><p>Creates immutable, date-scoped experiment Releases.</p><ul><li>Run-level digest comparison</li><li>Images/videos separated</li><li>Automatic asset partitioning</li><li>Supplemental Release handling</li></ul></article>
  <article class="tool-card"><code>tools/analyze_releases.py</code><p>Builds the canonical longitudinal dataset.</p><ul><li>Downloads metadata only</li><li>Normalizes runs and errors</li><li>Daily/monthly aggregation</li><li>Markdown, CSV, JSON, SVG, PNG</li></ul></article>
  <article class="tool-card"><code>tools/forecast_experiments.py</code><p>Runs the predictive intelligence layer.</p><ul><li>Rolling-origin model arena</li><li>Weighted ensemble</li><li>Residual-bootstrap intervals</li><li>10,000-path monthly simulation</li></ul></article>
</div>

## Generation runner

<Tabs>
  <TabItem label="Inputs">
  `agnes_media_config.yaml`, environment variables, and two JSONL prompt banks define models, quotas, intervals, output sizes, and prompt content.
  </TabItem>
  <TabItem label="Execution">
  Video and image phases submit jobs, poll asynchronous video status, classify provider/network errors, throttle requests, and stop a phase when policy rules require it.
  </TabItem>
  <TabItem label="Outputs">
  Each run contains `outputs.jsonl`, `errors.jsonl`, and `media/images` / `media/videos`. These files become the atomic input for publishing and analysis.
  </TabItem>
</Tabs>

## GitHub Actions

| Workflow | Trigger | Responsibility |
|---|---|---|
| `validate.yml` | PR and relevant pushes | Python compile/tests plus production site build |
| `analytics.yml` | Formal Release or manual dispatch | Ingest metadata, regenerate analytics, forecast, build and deploy Pages |
| `promote-input-snapshot.yml` | Manual dispatch | Reconstruct one snapshot, publish formal Releases, dispatch analytics |

<Aside type="tip" title="Adding another automated page">
Generate a JSON artifact after analytics, copy it into `web/public/data/`, add a component and MDX file, then register the page in `web/navigation.mjs`. The shell, sidebar, search, responsive behavior, and theme support remain shared.
</Aside>
