Technology responsibilities

Section titled “Technology responsibilities”
ASTRO

Static application compiler

Astro resolves the repository base path, turns every MDX document into a real directory route, bundles component scripts, fingerprints assets, and writes the final output to site/.

  • File-based routes under web/src/content/docs/
  • Build-time environment values such as BASE_URL
  • Minimal JavaScript outside interactive components
STARLIGHT

Documentation product shell

Starlight supplies the durable UX primitives that a hand-built page usually omits: responsive navigation, local search, dark and light themes, table of contents, semantic markup, keyboard behavior, and content typography.

  • Primary sidebar generated from the shared page registry
  • Reference pages for methodology and extension guides
  • Accessible defaults retained under custom visual styling
MDX + ASTRO COMPONENTS

Composable content layer

MDX keeps long-form documentation readable while allowing reusable interface elements such as the tab bar, metric strip, forecast dashboard, analytics dashboard, and interactive Mermaid viewer.

  • Documentation remains reviewable as text
  • Complex visual components stay isolated
  • New pages do not duplicate the site shell
ECHARTS

Analytical visualization engine

ECharts renders the high-density visual layer after JSON artifacts load. Canvas rendering keeps larger time series responsive while the toolbox adds data zoom, restore, legends, tooltips, and image export.

  • Daily and monthly throughput
  • Reliability and latency trends
  • Forecast intervals and model weights
MERMAID + PANZOOM

Executable architecture documentation

Mermaid compiles text diagrams into SVG. Panzoom is attached to the generated SVG so dense process maps support dragging, controlled wheel zoom, reset, and fullscreen inspection.

  • Strict Mermaid security mode
  • Theme-aware SVG generation
  • No hand-maintained diagram images
CUSTOM CSS

Product-specific visual system

The site extends rather than replaces Starlight. Shared tokens define panels, accents, spacing, charts, bento grids, tabs, mobile breakpoints, light mode, and reduced-motion behavior.

  • One visual language across docs and dashboards
  • Responsive five-tab navigation
  • System colors reused by ECharts and Mermaid

Why build-time URLs?

A data endpoint stored in a data-* attribute can be inspected in generated HTML and validated in CI. It also avoids relying on client-side assumptions about whether BASE_URL contains a trailing slash.

Data boundary

Section titled “Data boundary”
  • Analytics
  • Forecast
  • Architecture

AnalyticsDashboard.astro fetches analytics.json, then derives filtered headline metrics and renders daily, monthly, reliability, error, and run-ledger views. Release media is never downloaded by the browser.

ForecastDashboard.astro fetches forecast.json, then renders the next active-day ensemble, 80% intervals, operating regime, monthly Monte Carlo summaries, and model leaderboard.

MermaidDiagram.astro receives diagram source directly from MDX. Mermaid renders an SVG locally in the browser and Panzoom adds navigation controls without external services.

Build and deployment

Section titled “Build and deployment”

The CI validator checks that every registered primary route exists in the production output, that Overview links include the repository base, and that both dashboard components expose valid base-prefixed JSON artifact URLs. This makes the routing contract part of the build rather than a manual browser check.
