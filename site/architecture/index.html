System Atlas

The atlas describes the current repository as a four-plane system: generation, transport/storage, analytics/forecasting, and presentation.

Tool inventory

Section titled “Tool inventory”
agnes_media_harvester.py

Calls image/video APIs at conservative intervals and writes structured run metadata.

  • Loads YAML and prompt banks
  • Resumes from state
  • Writes outputs/errors JSONL
  • Downloads completed media
tools/publish_from_archive.py

ZIP-first ingress wrapper for Codespaces.

  • Validates ZIP64 structure
  • Rejects traversal and unsafe links
  • Checks disk capacity
  • Finds the results root
tools/input_snapshot.py

Stores a large input archive immediately and promotes it later.

  • Byte-exact 1.8 GiB splitting
  • Part/source SHA-256
  • Restore and promote modes
  • Latest-tag resolution
tools/publish_results.py

Creates immutable, date-scoped experiment Releases.

  • Run-level digest comparison
  • Images/videos separated
  • Automatic asset partitioning
  • Supplemental Release handling
tools/analyze_releases.py

Builds the canonical longitudinal dataset.

  • Downloads metadata only
  • Normalizes runs and errors
  • Daily/monthly aggregation
  • Markdown, CSV, JSON, SVG, PNG
tools/forecast_experiments.py

Runs the predictive intelligence layer.

  • Rolling-origin model arena
  • Weighted ensemble
  • Residual-bootstrap intervals
  • 10,000-path monthly simulation

Generation runner

Section titled “Generation runner”
  • Inputs
  • Execution
  • Outputs

agnes_media_config.yaml, environment variables, and two JSONL prompt banks define models, quotas, intervals, output sizes, and prompt content.

Video and image phases submit jobs, poll asynchronous video status, classify provider/network errors, throttle requests, and stop a phase when policy rules require it.

Each run contains outputs.jsonl, errors.jsonl, and media/images / media/videos. These files become the atomic input for publishing and analysis.

Adding another automated page

Generate a JSON artifact after analytics, copy it into web/public/data/, add a component and MDX file, then register the page in web/navigation.mjs. The shell, sidebar, search, responsive behavior, and theme support remain shared.
